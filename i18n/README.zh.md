## 这是什么

每天上新的东西太多，跟不完。这就是我的过滤器 —— 每天 3~5 条，各用一行说明为何值得一点。
不限主题：智能体、基础设施、小而巧的工具、模型，偶尔还有奇怪的实验。
每天的清单都冻结在归档里 —— 不会事后悄悄改写。

与所列项目无关。这里的条目是指路，不是推荐 —— 自己去看，自己判断。

## 今天 — 2026-07-23

潮流一直在说*去造一个新智能体*。今天的精选恰恰相反：给你已经在跑的那个智能体套上一层外壳 —— 编排它、压测它、接到画布上、指向某个领域。

| 精选 | 是什么 | 为何引起注意 |
|------|-----------|--------------|
| **T3MP3ST** | 驱动你现有编码智能体的自主红队元外壳 | 不自带模型 —— 它包住你已登录的智能体（Claude Code、Codex、完全离线的模型），跑一条侦察 → 利用 → 报告的杀伤链，用 egress-scope 隔离自动拒绝范围外主机；把红队重新定义为外壳问题，本月一路快速蹿升 |
| **Sim** | 搭建与编排智能体工作流的开源可视画布 | 赛道很挤，但 `npx simstudio` 一行即可自托管，整套东西本地接 Ollama、无需外部 API —— 正是这套一条命令、本地优先的叙事把它推上了 Product Hunt 首页 |
| **OpenScience** | 跑完整研究闭环的模型无关工作台 | 给它一个目标，它就读论文、提假设、在真实算力上写代码并运行、查询科学数据库、写出结果 —— 在浏览器的一个会话里连续完成，Apache-2.0、BYOK，并公开表明自己是 Claude Science 的独立替代 |
| **agent-drift** | 压测你的编码智能体会不会在压力下妥协 | 把 ICLR 2026「Asymmetric Goal Drift」的结论变得可运行：当约束与安全这类训练内化的价值相冲突时，智能体*更*容易违反系统提示，代码里普通的注释就足以随时间覆盖指令 —— 现在能在你自己的智能体上复现 |
