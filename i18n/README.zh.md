## 今天 — 2026-07-22

智能体的管道开始动真格。与其产出更多，今天的精选转向内部 —— 让智能体可观测、给它记忆、在自己的机器上运行、并指向安全。

| 精选 | 是什么 | 为何引起注意 |
|------|-----------|--------------|
| **mcpsnoop** | “MCP 版 Wireshark” —— 窥探工具调用的透明代理 | MCP Inspector 以自己的客户端接入、站在通道*旁边*；mcpsnoop 则切入真正的管道，让你终于看到生产客户端与服务器之间实际往来的 JSON-RPC、服务器 stderr，以及缓慢或卡死的调用 |
| **Strix** | 用于渗透测试的开源 AI 智能体 | 不用静态分析的误报，而是动态运行你的应用、用真实 PoC 验证每个发现，并把专门的智能体并行铺到各个目标 —— 把进攻变成一等的自主工作流 |
| **Juggler** | JUCE 作者打造的开源 GUI 编码智能体 | 逆着 CLI 智能体的潮流走图形化 UX —— 出自一位在 UX 上真有资历的老牌框架作者，等于在主张“终端不是编码智能体该待的地方”，值得关注 |
| **LM Studio Bionic** | 不是聊天窗，而是面向开源模型的完整本地智能体 | 越过聊天，能编辑文档、用内联 diff 跑智能体式代码检索、还做端上语音转写 —— 只把重活交给云端，数据零留存的本地优先 |
| **TencentDB-Agent-Memory** | 无需外部 API 的智能体本地长期记忆 | 四层本地流水线（本地 LLM + SQLite 向量）把工具日志凝练成紧凑符号，并将对话蒸馏为结构化的人物与场景 —— 而非一堆扁平向量 |
