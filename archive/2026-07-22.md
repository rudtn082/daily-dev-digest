# Daily Dev Digest — 2026-07-22

> Theme: The agent's plumbing gets serious — see it, give it memory, run it local, and let it break things safely.

## Picks

### mcpsnoop — "Wireshark for MCP," a transparent proxy for tool calls
The official MCP Inspector connects as its own client and sits *beside* the real channel, so it never sees what a production client and server actually say to each other. mcpsnoop drops into the actual pipe, surfacing every real JSON-RPC frame, server stderr, slow requests, and hung calls live in your terminal. As MCP wiring spreads, "what did my agent actually send" stops being a guessing game. Still pre-1.0, so behavior can shift on minor releases.

### Strix — open-source AI agents for penetration testing
Instead of the false positives of static analysis, Strix runs your app dynamically and validates each finding with a working proof-of-concept exploit. It arranges specialized agents into a graph, assigns them to different targets, and runs them in parallel so coverage widens as agents surface new paths. It's the pentesting counterpart to the coding-agent wave — offense as a first-class autonomous workflow, Apache-2.0.

### Juggler — a GUI coding agent from the creator of JUCE
Nearly every coding agent this year has been a CLI. Juggler bets the other way: a graphical UX for driving an AI code agent, from a veteran framework author who has spent a career on developer-facing tooling. Worth watching not because GUI-vs-terminal is settled, but because someone with deep UX credibility is arguing the terminal isn't the right home for this.

### LM Studio Bionic — a full local agent for open models, not just a chat window
LM Studio built its name on local model chat; Bionic pushes past that into an agent that edits documents, runs agentic code search with inline diffs against local folders, and does on-device voice transcription — routing only the heavy tasks to open models in the cloud. It's a bet that "local-first, you own the privacy and the spend" can carry real agentic work, not just conversation.

### TencentDB-Agent-Memory — fully local long-term memory for agents, zero external API
A four-tier pipeline (local LLM + SQLite vector search) that condenses noisy tool logs into compact symbols and distills fragmented conversations into structured personas and scenes, rather than dumping everything into a flat vector pile. Memory has been the missing local piece for agents that run for days; this treats it as a self-contained plugin you can host yourself.
