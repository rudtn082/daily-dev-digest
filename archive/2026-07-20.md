# Daily Dev Digest — 2026-07-20

> Theme: the anti-black-box day — instead of trusting a vendor runtime or logic that fires side effects out of sight, this batch hands you the source of the agent, keeps durable workflows on the Postgres you already run, makes I/O plain data you can read, and lets you host the edge platform yourself.

## Picks

### Grok Build — xAI's coding-agent CLI, open-sourced
xAI put the actual Rust source of its terminal coding agent on GitHub under Apache-2.0: the full-screen TUI, the agent runtime, tool execution, and the Agent Client Protocol wiring that lets editors embed it. It's a rare case of a frontier lab shipping the real harness behind its assistant rather than a thin wrapper around a hosted endpoint — so you can read exactly how context is assembled and tools are dispatched. The catch worth knowing up front: issues are disabled and outside contributions are rejected, so it's a source drop to learn from and run, not a community project to join.

### DBOSify — a drop-in Temporal replacement built on Postgres
Import `dbosify` instead of `temporalio`, point your clients and workers at a Postgres database instead of a Temporal server, and you get durable workflows, activities, signals, updates, retries, and crash recovery with no extra infrastructure to operate. It ports Temporal's own Python unit and integration tests to check conformance, so the compatibility claim is something you can inspect. There's no gRPC wire compatibility and non-Python SDKs can't connect, but for teams who adopted Temporal and now resent running the cluster, collapsing the whole thing down to a database they already have is a tempting trade.

### Pure Effect — business logic as plain data you can replay
A zero-dependency effects library for JS/TS where your functions return plain objects describing the I/O they *would* perform instead of performing it, and an interpreter runs those objects at the edge. Because the calls are captured rather than fired mid-logic, you can record what happened in production and replay the exact same path on your laptop with no database touched — turning "works on my machine" and stack-trace-only prod failures into something reproducible. It's an old idea (free monads, effect systems) packaged small and pragmatic enough that a normal codebase could actually adopt it.

### OpenWorkers — self-hosted Cloudflare Workers, in Rust
An open-source runtime that executes untrusted JavaScript in V8 isolates on your own servers, aiming for the Cloudflare Workers programming model without the vendor. It already covers a meaningful slice of the surface — `fetch`, KV, Postgres bindings, S3/R2, cron scheduling, and `crypto.subtle` — so a lot of real Worker code can run unchanged. As more teams weigh edge convenience against lock-in and egress bills, a hostable runtime that speaks the same DX is exactly the kind of escape hatch worth knowing exists.
