# Daily Dev Digest — 2026-07-21

> Theme: the agent stack grows up — instead of just generating more code and UI, today's tools put guardrails and coordination around what the models produce: gates against design slop, a git-native record of what an agent actually changed, and a runtime where agents self-organize instead of following a hand-drawn graph — plus a trillion-parameter open model trained off NVIDIA's silicon.

## Picks

### Hallmark — an anti-AI-slop design skill for Claude Code, Cursor, and Codex
Ask an agent to "build a landing page" and you know the result before you see it: the purple gradient, three identical feature cards, a hero that says "Revolutionize your workflow." Hallmark is a design skill that encodes rules refusing exactly those on-distribution defaults every frontier model was trained into, then runs a batch of "slop-test" gates plus a self-critique pass before it hands code back. It ships a set of built-in themes and a handful of verbs — build, audit, redesign, study — and drops to a from-scratch custom mode when a brief carries intent no catalog theme fits. It's a concrete answer to a complaint that's gone from meme to consensus this year: that AI-generated interfaces all look the same.

### Aura — a git-native provenance layer for AI coding agents
As agents write more of the diff, the reviewer's real question shifts from "does this compile" to "did it actually do what I asked, and nothing else." Aura parses your repo into an AST locally (Rust, Python, TypeScript, JavaScript) and tracks an agent's edits at the function and class level, so it can compare the change against the stated intent and block a commit when the two don't match. The free surface is unusually pointed: `aura rewind` reverts a single broken function instead of a whole checkpoint, an amnesia flag wipes an AI's memory of its own hallucinations, and a gatekeeper hook refuses commits where intent and AST modifications diverge. It's exposed as a CLI with git hooks, a desktop app, and a VS Code extension, under Apache-2.0.

### Mozaik — a TypeScript runtime where agents self-organize
Most agent frameworks make you draw the DAG up front: this node, then that node, then the branch. Mozaik makes a different bet — it models every message, tool call, reasoning step, and error as an event on a shared bus, and lets agents subscribe to what they care about and react as events arrive. Participants (agents, humans, tools, observers) join a shared "environment" and coordination emerges at runtime rather than being wired in advance, so multiple agents can work concurrently on the same goal without waiting on a fixed pipeline. For an existing TS project the first integration is small — install, define a few agents, connect them through an environment — which makes the event-driven model easy to try against the graph-first orthodoxy.

### LongCat-2.0 — a 1.6T open model trained entirely on domestic Chinese chips
Meituan open-sourced a 1.6-trillion-parameter Mixture-of-Experts model (roughly 48B active per token) under an MIT license — permissive is rare at this scale — and the headline isn't just the size. The company says it's the first model of this scale to complete both pre-training and inference entirely on domestic AI ASICs rather than NVIDIA GPUs, trained from scratch on a 50,000-card cluster. It ships a native 1M-token context made tractable by a custom linear-complexity attention scheme (LongCat Sparse Attention), an FP8 variant for deployment, and it has been near the top of OpenRouter as an agentic coding model. Whatever the benchmarks settle at, a permissively licensed trillion-parameter model trained off the NVIDIA grid is a supply-chain signal worth clocking.
