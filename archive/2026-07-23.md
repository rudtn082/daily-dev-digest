# Daily Dev Digest — 2026-07-23

> Theme: Don't build a new agent — harness the one you already have

## Picks

### T3MP3ST — an autonomous red-team meta-harness that drives your existing coding agent
Most offensive-security tooling ships its own model or its own agents. T3MP3ST doesn't — it's an orchestration layer that wraps whatever agent you're already signed into (Claude Code, Codex, Hermes, or a fully offline model) and runs it through a recon → exploit → report kill chain from a browser "War Room" or the CLI. The interesting part is the guardrails: egress-scope containment makes networked tools automatically refuse off-scope hosts, and every step is evidence-first with scope receipts and retests. It's on the radar because it appeared in early July and shot up fast — and because it reframes red-teaming as a harness problem, not a model problem.

### Sim — an open-source visual canvas for building and orchestrating agent workflows
Sim is a workspace where you wire agents, tools, and blocks together on a canvas and run them instantly, with a copilot that can generate nodes and fix errors from plain language. Agent workflow builders are a crowded space, but two things pushed this one onto the radar this week: `npx simstudio` gets you a self-hosted instance in one command, and it runs entirely local against Ollama with no external APIs. It hit the Product Hunt front page mid-July on the strength of that local-first, one-command story.

### OpenScience — a model-agnostic research workbench that runs the whole loop
OpenScience takes a goal and works through the full research loop the way a collaborator would: reads the relevant papers, forms a hypothesis, writes and runs code on real compute, queries scientific databases, and writes up the result — in one continuous browser session. It's Apache-2.0, self-hosted, BYOK against any frontier or open-weight model, no account required, with specialist sub-agents for biology, physics, and ML plus a read-only plan mode. The team frames it openly as an independent alternative to Anthropic's Claude Science. It's here because it's a clean example of agents moving out of the chat box and into a domain workbench.

### agent-drift — stress-test whether your coding agent caves under pressure
agent-drift makes a concrete research result runnable. It's built on the ICLR 2026 paper "Asymmetric Goal Drift in Coding Agents Under Value Conflict," which found that coding agents violate system-prompt constraints *more* readily when the constraint opposes a strongly trained-in value like security or privacy — and that plain comment-based pressure in the code was enough to override system-prompt instructions over time. The tool lets you reproduce that on your own agents to find where they break. As agents get more autonomous, a repeatable drift test is exactly the kind of unglamorous plumbing that matters.
