# Daily Dev Digest — 2026-07-16

> Theme: self-hosted and local-first tools keep gaining ground — own your stack, keep your data.

## Picks

### OpenKnowledge — open-source, AI-first alternative to Obsidian/Notion
A Notion-style space for writing and sharing markdown, built to plug into Claude/Codex more
directly than Obsidian does. Notable because the "AI-native notes" niche has mostly been closed
SaaS; an open, self-hostable take is overdue.

### Yao Agents — local-first agent execution platform (Go)
Runs agents on your own infra with Docker sandbox isolation, bring-your-own-key model config,
MCP support, and messaging across Telegram/Discord/Feishu and more. A serious self-hosted answer
to the cloud agent platforms.

### Observer AI — local automation agent that watches your system
An open control loop that observes your machine and acts on rules you define — local, no cloud
round-trip. Useful for people who want agent behavior without shipping their screen off-device.

### Prism.Tools — 40+ browser-based dev utilities, zero tracking
JSON formatter, regex tester, base64, git helpers and more, all running in the browser with no
data collection. A privacy-clean replacement for the ad-heavy "online tools" sites everyone
begrudgingly uses.

### BusterMQ — thread-per-core NATS server in Zig with io_uring
A from-scratch messaging server chasing low latency via a thread-per-core design and io_uring.
Worth watching as an example of the Zig + io_uring systems wave hitting real infra.
